# Insights And Notes Panel Redesign

## Context

首页问答链路已经重构为结构化学习时间线。当前第二批问题集中在“学习记录”和“卡片库”两页，它们的交互表达与当前产品方向不一致。

当前主要问题：

1. 学习记录页顶部四个概览卡只是展示，没有真实点击切换行为。
2. 学习记录页仍带有时间线式表达，而你明确要求把整套时间线删除。
3. 学习记录页缺少“概览卡点击后切换详情面板，同时保留原列表”的结构。
4. 卡片库里的“进入复习”按钮位置较弱，在卡片底部，不够明确。
5. 卡片库的“进入复习”按钮视觉不够突出，需要改成深色主按钮。

本轮只处理 `学习记录` 与 `卡片库` 两页，不再调整首页问答流。

## Goals

### 学习记录页

1. 删除整套时间线结构，不保留竖线、节点或连接线。
2. 顶部四个概览卡变成真实可点击状态。
3. 点击任一卡片后，中段切换成对应详情面板。
4. 详情面板切换后，下方原记录列表仍保留。
5. 记录列表改成普通内容列表，不再使用时间线表现。

### 卡片库页

1. 将“进入复习”按钮移动到卡片内部上方。
2. 按钮改为深色主按钮。
3. 点击行为保持不变，仍在右侧详情区展开 `ReviewMode`。

## Non-Goals

本轮不做：

1. 不重构首页问答。
2. 不引入新的后端接口。
3. 不重构卡片库整体双栏布局。
4. 不修改 ReviewMode 的内部学习阶段逻辑。
5. 不把 Notes / Insights 迁移到真实后端数据源。

## User Experience

## Learning Records Page

### Top Overview Cards

顶部四张卡片成为“分类入口”，不再是静态信息卡。

点击后行为：

1. 当前卡片进入 active 状态
2. 中段详情面板切换为对应类别内容
3. 下方记录列表保留原结构，但仍可以继续浏览全部记录

### Middle Detail Panel

中段详情面板用于展示当前选中卡片对应的主题摘要，包括：

1. 当前类别标题
2. 简短说明
3. 核心代表内容
4. 对应的学习建议或复习入口

详情面板是切换式面板，不是弹窗，也不是跳页。

### Records List

下方记录区域不再以时间线形式表达。

改为普通纵向列表卡片：

1. 每条记录独立成块
2. 不显示连线、节点、轴线
3. 每条记录仍可触发“重新复习”

## Notes Page

### Review Button Placement

“进入复习”按钮调整为：

1. 放到卡片内部上方区域
2. 与标题、标签信息形成更明确的操作区
3. 视觉改为深色主按钮，明显高于辅助信息

### Review Behavior

点击后行为不变：

1. 选中当前卡片
2. 右侧详情区展开 `ReviewMode`

## Visual Design

### Learning Records

学习记录页整体改成“概览 + 面板 + 列表”的结构。

视觉要求：

1. 顶部四卡有明确 active 状态
2. 中段详情面板比列表更突出，但不厚重
3. 去除时间线装饰元素后，列表留白更清晰

### Notes

卡片库中的复习按钮要：

1. 使用深色背景
2. 保持圆角统一
3. 与卡片标题区对齐
4. 不再放在卡片底部弱操作区

## Information Architecture

### Insights

结构变为三层：

1. 顶部概览卡切换层
2. 中段详情面板层
3. 下方记录列表层

### Notes

保持双栏结构：

1. 左侧列表
2. 右侧详情

只调整左侧卡片内部的按钮层级与位置。

## Data Model

### Insights

新增页面级状态：

1. `activeOverviewKey`
2. `activePanelContent`

用于驱动：

1. 顶部四卡 active 状态
2. 中段详情面板内容切换

### Notes

不新增新的业务模型，只调整 `NoteCard` 的展示结构和按钮位置。

## Error Handling

本轮不涉及新的远程请求，因此无需新增网络错误处理。

若详情面板找不到对应数据：

1. 回退到默认类别
2. 不影响下方列表展示

## Implementation Scope

主要涉及：

1. `frontend/src/pages/InsightsPage.vue`
2. `frontend/src/components/insights/*`
3. `frontend/src/pages/NotesPage.vue`
4. `frontend/src/components/notes/NoteCard.vue`
5. 若需要，补充轻量的 mock panel data 文件

## Acceptance Criteria

完成后应满足：

1. 学习记录页不再有任何时间线视觉元素
2. 顶部四卡可点击并切换 active 状态
3. 点击后中段详情面板会切换内容
4. 下方记录列表仍保留
5. 每条记录仍支持“重新复习”
6. 卡片库中“进入复习”按钮移到卡片内部上方
7. 该按钮改为深色主按钮
8. 点击后仍在右侧详情区展开 ReviewMode

## Recommendation

按“概览卡 + 内容面板 + 普通记录列表”的方式重做学习记录页，同时只对卡片库进行最小但明确的复习按钮改造。
