# 官网首页设计稿：Learning Capsule

日期：2026-04-08
范围：营销官网首页
状态：设计已确认，待用户 review 后进入实现计划

## 1. 目标

为 Law Assistant 设计一个营销型官网首页，核心目标是：

- 在 5 到 8 秒内让访客理解产品价值
- 展示产品的核心学习能力，而不是把首页做成普通问答页
- 用强 CTA 引导用户点击“开始学习”
- 建立“年轻学习产品感”为主、混合“专业可信赖”的品牌气质

本页不是应用内工作台，不承载完整功能闭环。它是品牌入口页、试用转化页、产品价值展示页。

## 2. 视觉方向

首页采用 “Learning Capsule / 学习宇宙舱” 方向。

视觉关键词：

- 年轻
- 清爽
- 学习科技感
- 轻动态
- 可信赖
- 不律所模板化

设计原则：

- 参考 Aceternity UI 的页面表达方式，而不是逐字逐组件照搬 React 实现
- 在 Vue 3 + Tailwind 中重建其视觉语言与交互动势
- 用浅色背景、渐变光感、柔和网格与悬浮层次营造首页记忆点
- 不使用老派法律网站常见的厚重蓝金配色和证章式排布

整体基调：

- 主色：偏青绿的学习科技色
- 辅色：深墨灰文本、浅雾白底、少量偏蓝高光
- CTA：强调“开始学习”

## 3. 参考实现方式

本页将借鉴 Aceternity UI 的以下表达方式：

- Hero Section：用于首屏记忆点与主转化区
- Background Effects：用于首屏氛围与空间感
- Bento Grid：用于能力模块展示
- Stats / Testimonial 风格区块：用于补充信任和完成度
- Card Motion / Spotlight 风格：用于少量高价值 hover 反馈

这里的目标不是引入 React 组件，而是学习其：

- 信息排布
- 光效与背景语言
- 模块节奏
- 视觉重心控制
- 动效层次

## 4. 页面结构

首页建议分为 6 个区块。

### 4.1 Hero 首屏

目标：

- 在第一屏完成产品理解与 CTA 收束

内容：

- 品牌标题与导航
- 主标题
- 副标题
- 主按钮：开始学习
- 次按钮：查看演示
- 右侧或下方产品界面预览

文案方向：

- 主标题强调“把法律问题拆成可学习的结构”
- 副标题强调“概念、法条、案例、易混点、笔记沉淀”

视觉要求：

- 使用轻动态背景
- 中心标题清晰但不过度硬核
- 页面第一记忆点来自“AI 学习工作台感”，而不是插画本身

### 4.2 核心能力 Bento Grid

目标：

- 快速展示产品不是普通聊天机器人

模块建议：

- 问题结构化拆解
- 法律概念快速理解
- 典型案例辅助记忆
- 易混概念对比
- 一键保存为笔记
- 持续复习与积累

视觉要求：

- 使用不等尺寸模块
- 通过节奏变化建立阅读流
- 每个模块都像产品能力卡片，而不是介绍文字堆

### 4.3 三步学习流程区

目标：

- 让用户理解产品的最小闭环

步骤：

1. 提出法律问题
2. 获得结构化学习结果
3. 保存为笔记并持续复习

视觉要求：

- 不做厚重流程图
- 更像轻量产品导览
- 允许加入 unDraw 插画作为辅助说明

### 4.4 适用人群 / 场景区

目标：

- 帮用户快速代入

适用对象：

- 法学生
- 法考备考者
- 法律初学者

视觉要求：

- 使用真实照片增强可信度
- 排版保持年轻感，不做传统“目标客户列表”

### 4.5 产品预览区

目标：

- 展示产品已经具备真实界面与功能，不只是概念页

内容：

- 首页工作台预览
- 结构化学习结果预览
- 笔记页预览

视觉要求：

- 可以使用真实页面截图或模拟 UI
- 重点展示产品感和完成度

### 4.6 底部 CTA

目标：

- 统一收束到“开始学习”

内容：

- 一句简洁召唤文案
- CTA 按钮

## 5. 素材策略

按资源来源分配如下：

### 5.1 插画

来源：unDraw

使用位置：

- 三步学习流程区
- 辅助解释学习场景

要求：

- 选择与学习、思考、知识整理相关的插画
- 不选择与法律职业过于传统的法槌、天平滥用插画

### 5.2 图标

来源：Iconify

使用位置：

- 能力模块
- 流程区
- 场景区标签

要求：

- 图标风格统一
- 线性图标优先
- 避免 emoji 风格

### 5.3 真实照片

来源：Pexels

使用位置：

- 场景区
- 可信度补充区

要求：

- 优先年轻学习、阅读、电脑学习场景
- 不使用过于商务化或律师事务所会议室照片

### 5.4 占位图

来源：Picsum Photos

仅在找不到合适真实素材时临时使用，不作为最终主视觉资源。

## 6. 组件与实现建议

在现有 Vue 3 + Tailwind 项目中，官网首页建议拆成以下结构：

- `pages/MarketingHomePage.vue`
- `components/marketing/HeroSection.vue`
- `components/marketing/FeatureBentoGrid.vue`
- `components/marketing/LearningFlowSection.vue`
- `components/marketing/AudienceSection.vue`
- `components/marketing/ProductPreviewSection.vue`
- `components/marketing/BottomCtaSection.vue`

如果当前路由结构允许，也可以先以单页组件方式实现，再视情况拆分。

## 7. 动效策略

动效必须克制且有目的：

- Hero 首屏背景有轻微流动感
- 能力模块 hover 有精细反馈
- 页面加载有分层渐入
- CTA 有清晰但不夸张的交互反馈

禁止：

- 过度花哨的粒子特效
- 持续强运动影响阅读
- 让整页像模板站或 crypto 风格 landing page

## 8. 响应式要求

- 桌面端：完整呈现 Hero + Bento + 场景区
- 平板端：保持层级但缩减横向复杂度
- 移动端：Hero、CTA、能力模块优先展示
- 所有页面禁止横向滚动
- 保持点击区尺寸和阅读舒适度

## 9. 验收标准

当满足以下条件时，首页第一轮可视为完成：

- 首屏明确表达产品价值且 CTA 清晰
- 页面整体气质符合“年轻学习产品感 + 一点专业可信赖”
- 明显能看出产品不是普通聊天机器人
- 使用 Aceternity UI 风格语言，但能自然融入当前 Vue 项目
- 已整合插画、图标、照片等合适素材
- 桌面和移动端都具备良好可读性与完成度

## 10. 风险与控制

风险：

- 过度追求酷炫，削弱可信度
- 页面营销化太强，和产品真实体验脱节
- 首页元素过多导致转化分散

控制方式：

- 统一收束到“开始学习”
- 所有展示能力必须和真实产品结构对齐
- 保持年轻感，但用排版、留白和真实产品预览维持专业感
