# 法小智 AI 法律学习工作台设计

## 背景

当前前端项目仍停留在单组件演示状态，缺少完整产品骨架，也还没有形成“法律学习工作台”应有的信息层级和页面节奏。

这次重建的目标不是做一个聊天机器人界面，而是搭建一个完整的前端学习型产品原型，让用户感受到：

- 这是一个结构化学习平台
- 这是一个法律知识整理与沉淀空间
- 这是一个干净、现代、可信赖的 SaaS 产品

实现范围只包含前端 UI 和本地 mock 交互，不接入后端，不实现真实 API。

## 目标

基于 Vue 3、Composition API 和 TailwindCSS，搭建一个完整的小型前端项目，包含：

- 主应用布局
- 可折叠侧边栏
- 三个具有明显身份差异的页面
- 清晰的组件结构
- 本地 mock 数据驱动的页面内容和交互

三页分别为：

- 首页问答
- 卡片库
- 学习记录

## 非目标

以下内容不在本次范围内：

- 后端接口联调
- 用户登录鉴权
- 聊天气泡对话流
- 真正的 AI 推理与答案生成
- 复杂全局状态管理

## 产品定位

产品名为“法小智”。

定位不是“你问我答”的通用聊天工具，而是“围绕法律知识点进行理解、拆解、比较、记录与复习”的学习工作台。

因此所有界面设计都要围绕“学习单元”和“知识沉淀”展开，而不是围绕消息流展开。

## 设计方向

整体视觉方向采用“现代 SaaS 学习平台”为主，混入少量“书房式学习工作台”的温度感。

具体表现为：

- 以浅灰蓝和暖白为主色基底
- 页面背景使用 `#F7F8FA`
- 卡片使用白底、细描边、轻阴影、大圆角
- 保持安静、克制、清晰的视觉节奏
- 用少量偏暖中性色缓和纯 SaaS 的冷感

界面必须避免：

- ChatGPT 式聊天界面
- 大面积深色块
- 重后台仪表盘风格
- 过于活泼或玩具感的视觉语言

## 技术方案

采用标准单页应用结构。

- 使用 `vue-router` 管理三页导航
- 使用 `MainLayout` 承载侧边栏与主内容区
- 页面通过组合式 API 管理本地交互状态
- 所有内容来自 `data/` 下的 mock 数据文件
- 不引入 Pinia 等状态库

选择 `vue-router` 的原因：

- 更接近真实产品结构
- 支持三页独立路径与后续扩展
- 组件边界清晰，不会把整个应用写成一个大视图切换器

## 信息架构

### 全局框架

整体为“左侧边栏 + 右侧主内容区”。

不设置重型全局顶部导航。每页自己的标题和功能区在主内容中完成。

### 侧边栏

侧边栏默认展开，支持折叠：

- 展开宽度：`240px`
- 折叠宽度：`72px`
- 折叠后仅显示图标
- 折叠状态下悬停显示 tooltip
- 当前激活导航项高亮

侧边栏内容：

- Logo / 产品名
- 首页问答
- 卡片库
- 学习记录
- 主题切换
- 用户信息区

视觉要求：

- 浅色底
- 极轻边界和阴影
- 不做暗黑侧栏
- 不抢首页学习区的视觉重心

## 页面设计

### 页面一：首页问答

首页是产品最重要的页面，承担从“进入学习”到“结构化理解”的主流程。

该页不保留右侧辅助栏，只保留左侧边栏与中心主区，让用户更聚焦在学习入口和结果模块上。

### 页面身份

- 三页中最有呼吸感
- 最强调输入和学习结果
- 布局居中、节奏克制
- 不出现聊天记录堆叠

### 页面状态

首页有两个主要状态：

1. 初始状态
2. 学习结果状态

### 初始状态结构

顺序如下：

1. Welcome 区
2. Continue learning 卡片
3. Recommended topics
4. 主问题输入框
5. Example question chips

#### Welcome 区

展示：

- 产品名：法小智
- 描述：帮助你真正理解并掌握法律知识

风格要求：

- 紧凑而优雅
- 不做夸张 hero
- 更像工作台开场而不是营销落地页

#### Continue learning 卡片

展示最近学习主题，例如：

- 合同法 · 违约责任
- 上次学习：昨天
- 按钮：继续学习

作用是让用户感受到这是一个“可持续学习”的产品，而非一次性问答工具。

#### Recommended topics

展示 3 张推荐主题卡：

- 不安抗辩权
- 缔约过失责任
- 善意取得

交互规则：

- 点击任意主题，进入结果状态
- 当前问题替换为该主题对应的问题标题

#### 主问题输入框

采用独立组件 `QuestionInput`。

要求：

- 默认两行 textarea
- 自动扩展至四行
- placeholder：`输入法律问题，例如：什么是不安抗辩权`
- 主按钮文本：`解析`

视觉定位：

- 首页的核心动作组件
- 比结果态底部的追问输入框更强

#### Example question chips

展示 4 个可点击问题标签：

- 什么是不安抗辩权
- 违约责任构成要件
- 举一个合同违约案例
- 抗辩权之间的区别

交互规则：

- 点击标签可填充输入框
- 可直接触发结果态

### 结果状态结构

结果态使用学习模块而不是对话气泡。

顺序如下：

1. 问题标题卡
2. 结构化学习模块
3. 学习动作按钮
4. 底部追问输入框

并提供一个轻量“重新提问”或“返回探索”入口，让用户可以切回首页初始态。

#### 问题标题卡

清晰展示当前问题，例如：

`什么是不安抗辩权`

这个卡片应作为当前学习主题的总入口，而不是消息条。

#### 结构化学习模块

使用独立卡片展示以下模块：

- 概念解释
- 构成要件
- 案例
- 常见误区
- 相关法条
- 易混概念

每张卡片包含：

- 图标
- 标题
- 内容

内容形式支持：

- 简短段落
- 编号列表
- 高亮提示

模块区应像“法律学习单元系统”，而不是聊天回复拆段。

#### 学习动作按钮

结果区下方展示一排学习动作：

- 再简单一点
- 举个案例
- 对比易混概念
- 生成学习笔记

设计要求：

- 视觉更像学习操作，不像快捷回复
- 可点击后替换或聚焦相关 mock 内容
- 不渲染为气泡按钮堆

#### 底部追问输入框

用于继续深入当前知识点。

视觉上弱于首页首次输入框，但仍保持清晰可用。

## 页面二：卡片库

卡片库是知识积累页面，强调秩序、稳定和可阅读性。

### 页面身份

- 三页中最稳定
- 最强调“筛选 + 浏览 + 阅读”
- 像个人法律知识卡片库，而不是后台列表

### 布局

采用双栏布局：

- 左栏：搜索、筛选、排序、布局切换、卡片列表
- 右栏：笔记详情阅读区

右侧详情区是主阅读区域，不能做成狭窄小侧栏。

### 左栏结构

依次包含：

1. 搜索输入
2. 标签筛选
3. 排序控制
4. 布局模式切换
5. 笔记卡片列表

#### 搜索输入

placeholder：`搜索笔记`

#### 标签筛选

mock 标签：

- 合同法
- 侵权法
- 物权法
- 民法典

支持单选或多选的本地状态筛选，首版以单选实现即可。

#### 排序控制

选项：

- 最近学习
- 创建时间
- 重要程度

#### 布局模式切换

两种模式：

- Grid
- Masonry

要求：

- 默认使用 Grid
- Masonry 依然要规整、易读，不能故意做乱
- 使用 CSS columns 或平衡列实现即可

#### 笔记卡片列表

每张卡片展示：

- 标题
- 摘要
- 标签
- 最近学习时间
- 要点数
- 当笔记状态为待复习时，显示“待复习”徽标

选中态要求：

- 清晰可见
- 不要重色块压住内容
- 用描边、背景色和阴影轻微增强即可

### 右栏详情结构

显示当前选中的笔记详情，包含：

- 标题
- 编辑按钮
- 查看详情按钮
- 时间 / 分类
- 来源问题
- 结构化摘要
- 关键要点
- 学习建议
- 标签

详情区的阅读体验应接近专题学习页。

## 页面三：学习记录

学习记录页应当像“个人学习反思页”，而不是报表中心。

### 页面身份

- 比首页更分析化
- 比卡片库更柔和
- 更强调反馈、诊断和节奏

### 页面结构

1. 顶部洞察概览
2. 学习诊断
3. 学习时间线

#### 顶部洞察概览

4 张轻量卡片：

- 今日学习
- 本周学习
- 累计掌握
- 待复习

每张卡片包含：

- 数字
- 一句解释性描述

数字不是唯一主角，文案和语气要让用户感受到“学习进展”。

#### 学习诊断

这是价值最高的区域，包含：

- 最近 7 天学习趋势
- 最近最容易卡住
- 建议重点区分

##### 学习趋势

使用轻量折线图样式，采用 SVG 或纯 CSS 绘制即可。

mock 内容表示最近 7 天学习次数变化。

##### 最近最容易卡住

展示易混或常卡主题：

- 不安抗辩权
- 善意取得
- 代位权

##### 建议重点区分

展示概念对：

- 先履行抗辩权 vs 不安抗辩权
- 缔约过失责任 vs 违约责任

#### 学习时间线

展示最近学习活动流，每条包含：

- 主题
- 学习结果
- 时间

例如：

- 违约责任构成要件 / 已掌握 / 今天 14:20
- 不安抗辩权 / 待复习 / 昨天 20:10

风格更接近学习轨迹，而不是表格或日志表。

## 交互规则

所有交互都使用本地 mock 状态完成。

### 全局

- 侧边栏支持展开/折叠
- 折叠态导航有 tooltip
- 主题切换先做视觉开关，不要求完整 dark mode 体系

### 首页

- 点击推荐主题进入结果态
- 点击示例问题可填充输入并进入结果态
- 点击学习动作按钮可替换当前聚焦模块内容或更新当前问题
- 支持在初始态与结果态之间切换

### 卡片库

- 点击左侧笔记卡片，右侧详情更新
- 标签和搜索只过滤 mock 数据
- 布局模式在 grid 和 masonry 间切换

### 学习记录

- 图表和诊断内容全部来自 mock 数据
- 时间线按最近活动排序展示

## 视觉规范

### 背景

- 全局背景：`#F7F8FA`

### 卡片

- 白底
- `rounded-xl` 或 `rounded-2xl`
- 细边框
- 很轻的阴影
- 充足内边距

### 字体层级

- 页面标题：大、稳、清晰
- 区块标题：中等字号和较强字重
- 正文：高可读性
- 元信息：更浅、更小

### 页面差异化

#### 首页

- 留白最多
- 最中心化
- 输入和学习结果是绝对主角

#### 卡片库

- 结构最稳定
- 浏览与阅读分区最明确
- 更强调秩序感

#### 学习记录

- 视觉更柔和
- 分析感弱于后台
- 更像个人学习反馈页

## 数据与实现策略

mock 数据文件按页面拆分，包含：

- 首页推荐主题、继续学习信息、学习模块内容
- 卡片库笔记列表与详情
- 学习记录概览、趋势、困惑主题、活动流

实现时先构建布局和页面骨架，再逐页补齐组件和 mock 交互，最后统一补足全局样式和细节 polish。

## 验收标准

完成后的前端应满足：

- 打开后是完整三页产品，而不是单组件 demo
- 视觉上明显不是聊天机器人界面
- 三个页面有不同身份和节奏
- 侧边栏折叠、首页状态切换、卡片详情切换、布局模式切换都可用
- 组件结构清晰，可继续扩展
- 仅依赖 mock 数据即可完整浏览主要流程
