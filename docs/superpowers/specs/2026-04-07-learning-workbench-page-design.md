# 法小智学习工作台页面设计

## 背景

当前产品已经明确从“泛法律问答工具”转向“理解型法律学习产品”。  
核心价值不再是像聊天机器人一样回答问题，而是帮助用户围绕一个法律知识点完成理解、追问、总结和沉淀。

现有三页已有初步代码骨架：

- 首页问答
- 笔记库
- 学习记录

但当前页面仍有明显的传统 AI 对话页和后台页倾向，尚未形成统一的产品界面语言。因此需要先定义一套统一的页面设计逻辑，再进入后续视觉和实现阶段。

## 目标

将产品整体页面统一为“学习工作台”风格，并确保：

- 首页问答是最核心、最聚焦的页面
- 笔记库是最稳定、最清晰的知识管理页面
- 学习记录是更偏洞察和反馈的页面，而不是后台报表
- 全站不强调聊天气泡，而强调结构化学习模块
- 首页左导航和右侧学习辅助区支持折叠

## 设计原则

### 产品层原则

- 产品形态应是“学习工作台”，不是“大模型聊天界面”。
- 交互核心是“学习一个知识点”，不是“连续聊天”。
- 页面重点是结构化理解、知识沉淀和学习反馈，而不是消息往返。

### 视觉层原则

- 整体明快、干净、留白充足。
- 页面采用轻卡片布局，但卡片不厚重、不做强后台感。
- 层级主要依靠留白、间距、标题层级和轻边框，而不是大面积重色块。
- 图标用于强化学习语义，不作为强装饰元素。

### 布局层原则

- 首页问答最强，中间主区最大。
- 笔记库最稳定，三栏结构最清楚。
- 学习记录更像个人学习洞察页，不做重后台感。
- 右侧辅助栏只在首页强调，在另外两页中弱化或移除。

## 全局框架

全局采用“左导航 + 主内容区”的学习工作台结构。

### 左侧导航

默认展开，支持折叠。

展开状态建议包含：

- 产品名 / Logo
- 首页问答
- 笔记库
- 学习记录
- 主题切换
- 用户入口

折叠状态规则：

- 宽度由约 `240px` 收窄至约 `72px`
- 保留图标
- 悬停时显示名称

导航设计应保持轻量，不应抢占首页主学习区的注意力。

## 页面一：首页问答

首页问答是整个产品最重要的页面，承担“输入问题 -> 结构化理解 -> 继续追问 -> 总结沉淀”的核心流程。

### 页面结构

采用三段式布局：

- 左侧可折叠导航
- 中间主学习区
- 右侧可折叠辅助区

其中中间主学习区权重最高。

### 顶部区

首页不采用传统厚重 header，而是轻量页头。

建议内容：

- 页面标题或产品标题
- 轻量状态提示
- 左导航折叠按钮
- 右辅助区折叠按钮

顶部区应服务于页面组织，不应喧宾夺主。

### 中间主学习区

主学习区分为两种状态。

#### 初始态

面向第一次进入或未发起学习时的状态。

模块包括：

- 品牌区
  - 法小智
  - 一句话定位
- 大输入框
- 开始学习按钮
- 建议问题区

设计要求：

- 初始态内容垂直居中偏上
- 输入框是视觉焦点
- 建议问题以轻量标签按钮形式展示
- 不出现聊天历史堆叠

#### 结果态

用户输入问题后，页面进入结果态。

模块顺序为：

- 问题标题卡
- 结构化学习卡片组
- 引导式按钮区
- 底部继续追问输入框

结果态不采用聊天气泡，而采用结构化学习模块。

### 结构化学习卡片组

根据当前代码，首页结构化学习卡片至少包括：

- 概念分析
- 构成要件
- 案例举例
- 常见误区

后续可扩展：

- 关联法条
- 易混概念
- 民法请求权基础分析
- 刑法四要件

卡片设计要求：

- 每张卡片作为独立学习模块存在
- 采用轻边框、轻底色或纯白底卡片
- 标题、图标、正文层级清楚
- 允许重点信息高亮

### 引导式按钮区

首页的差异化重点在于“引导式学习”。

引导按钮放在结构化结果之后，用来推动用户继续理解，而不是让用户自己决定下一步。

当前固定按钮包括：

- 再简单一点
- 举个例子
- 对比易混概念
- 总结成笔记

设计要求：

- 视觉上接近“学习操作按钮”，而非聊天快捷回复
- 尺寸适中，贴近结果区
- 首版可固定，后续可根据回答动态变化

### 底部继续追问区

底部提供单独的继续追问输入框，用于多轮学习。

该输入区是“继续推进理解”的入口，不是新一轮独立对话入口，因此应在视觉上弱于首轮输入框。

### 右侧辅助区

首页保留右侧辅助区，并支持折叠。

当前结构可保留为：

- 今日学习
- 最近聊天
- 当前学习建议

作用定位：

- 辅助学习
- 快速回到近期上下文
- 展示当前学习状态

折叠规则：

- 默认展开
- 用户可主动收起
- 收起后中间主学习区自动扩展

## 页面二：笔记库

笔记库承担知识沉淀和知识浏览的作用，应是全站最稳定、最秩序化的一页。

### 页面结构

采用稳定的三栏结构：

- 左侧筛选区
- 中间笔记列表区
- 右侧笔记详情区

这页强调的是：

- 查找
- 浏览
- 深读

### 左侧筛选区

根据当前代码保留如下模块：

- 搜索框
- 分类筛选
- 常用标签

要求：

- 宽度稳定
- 信息密度适中
- 分类和标签不应视觉过重

### 中间笔记列表区

中间区域承载笔记浏览。

保留模块：

- 页面标题“我的笔记”
- 新建笔记按钮
- 笔记卡片列表

每条笔记卡片保留信息：

- 标题
- 分类
- 摘要
- 标签
- 最近学习时间
- 要点数量

设计要求：

- 列表结构清晰
- 选中态明确但不过重
- 像“知识卡片列表”，而不是“消息列表”或“文件管理器”

### 右侧笔记详情区

右侧详情区是笔记精读和再加工区域。

保留模块：

- 标题与编辑入口
- 时间与分类
- 结构化总结
- 核心要点
- 学习建议
- 标签区

设计要求：

- 右侧详情区宽度充足
- 内容层级清楚
- 更像阅读与整理页面，不像弹窗详情页

### 页面定位

笔记库页的核心逻辑是：

`筛选 -> 浏览 -> 精读`

因此这页的稳定性、秩序性和信息清晰度优先于动态感。

## 页面三：学习记录

学习记录页的目标不是做成后台统计页，而是做成“个人学习洞察页”。

### 页面结构

采用三段结构：

- 顶部洞察总览
- 中部诊断分析
- 底部学习轨迹

### 顶部洞察总览

保留当前四个核心指标：

- 今日学习
- 本周学习
- 累计掌握
- 连续学习

但表达方式要去后台化。

要求：

- 卡片更轻
- 除数字外增加简短解释
- 更像“你最近的学习状态”

### 中部诊断分析

这是学习记录页最有价值的部分。

当前可保留：

- 本周学习趋势
- 高频卡点类型
- 容易混淆的概念对

但要从“图表展示”转向“学习诊断”表达。

例如：

- 高频卡点应强调“你最近最容易卡住的内容”
- 易混概念应强调“建议重点区分的概念对”
- 趋势图应更柔和，不像硬后台报表

### 底部学习轨迹

底部用来展示最近学习内容。

每条轨迹建议表达：

- 学习行为类型
- 学习内容
- 学习结果
- 时间

这部分应更像“学习轨迹流”，而不是单纯活动列表。

### 页面定位

学习记录页的核心逻辑是：

`最近学了什么 -> 最近卡在哪里 -> 接下来建议怎么学`

## 页面差异化要求

为了避免三页看起来像同一个后台模板，必须明确区分。

### 首页问答

- 中间主区最大
- 右侧辅助区保留
- 页面呼吸感最强
- 最强调输入、结构化输出和引导式学习

### 笔记库

- 三栏结构最稳定
- 最强调秩序和浏览效率
- 最像知识资产页

### 学习记录

- 最强调洞察与反馈
- 降低后台统计感
- 更像个人学习状态页

## 折叠规则

### 首页左导航

- 默认展开
- 用户可手动折叠
- 折叠后主区自动变宽

### 首页右辅助区

- 默认展开
- 用户可手动收起
- 收起后主区自动扩展

### 笔记库

- 左侧筛选区可保持稳定
- 右侧详情区保持常驻

### 学习记录

- 不设置常驻右侧辅助区
- 主区单独承担页面内容

## 当前结论

当前版本页面设计应明确遵循以下统一逻辑：

- 首页问答最强，中间主区最大
- 笔记库最稳定，三栏最清楚
- 学习记录最像洞察页，不做重后台感
- 全部页面都用卡片化布局，但卡片尽量轻
- 不强调对话气泡，强调学习模块
- 右侧栏在首页保留，在另外两页可以弱化或隐藏
- 首页左导航和右侧学习辅助区支持折叠
